# Features Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement MVP feature modules in `general`, `owner`, and `group` categories.

**Architecture:** Flat files by default; folder modules only when needed. Shared features must use `MessageCtx` and `AppContext`, not native platform clients.

**Tech Stack:** Contracts, core guards, Prisma repos, Vitest `createMockCtx()`.

---

## Files

- Create: `packages/features/src/general/{ping,stats,help,menu}.ts`
- Create: `packages/features/src/general/remind/{index,_handlers,_subscriptions}.ts`
- Create: `packages/features/src/owner/{eval,broadcast,shutdown}.ts`
- Create: `packages/features/src/group/{kick,mute,antilink,welcome}.ts`
- Modify: `packages/features/src/index.ts`
- Tests per category

## Task 1: General basics

- [ ] Implement `general/ping.ts`: reply `pong (latency: Xms)` using `Date.now() - ctx.timestamp`.
- [ ] Test ping with fixed timestamp.
- [ ] Implement `general/stats.ts`: uptime, RSS memory, node version, feature count, command count, DB status.
- [ ] Test stats includes `uptime`, `memory`, `commands`.
- [ ] Implement `general/menu.ts`: text-only menu grouped by General/Owner/Group.
- [ ] Test menu visibility for owner vs non-owner.

## Task 2: Help

- [ ] Implement `general/help.ts` with `!help` and `!help <cmd>`.
- [ ] Hide owner commands and aliases for non-owner.
- [ ] Hide group commands in DM context.
- [ ] Test non-owner does not see owner command names or aliases.
- [ ] Test `!help ping` shows description, usage, aliases, category.

## Task 3: Remind folder feature

- [ ] Implement `general/remind/index.ts` exporting commands `remind`, `reminders`, `cancelreminder` alias `cancel`, and event `reminder.fire`.
- [ ] Implement `_handlers.ts`: create reminder, list pending reminders, cancel only own reminder.
- [ ] Implement `_subscriptions.ts`: on fire, send via `app.adapters.get(platform).sendMessage()`, then mark done/failed.
- [ ] Test create/list/cancel/fire with mocked DB and adapter registry.

## Task 4: Owner features

- [ ] Implement `owner/eval.ts` using `vm.runInNewContext` with timeout and 4000-char output cap.
- [ ] Implement `owner/broadcast.ts` loading known users/groups and sending via adapter registry.
- [ ] Implement `owner/shutdown.ts` invoking shutdown hook through app context contract or event.
- [ ] Test exported features do not manually include `requireOwner`; loader injects it.

## Task 5: Group features

- [ ] Implement `group/kick.ts`: attempt platform kick via adapter capability if available; otherwise reply unsupported/admin-required message.
- [ ] Implement `group/mute.ts`: toggle `GroupConfig.muted`.
- [ ] Implement `group/antilink.ts`: toggle antiLink and middleware/event URL delete behavior.
- [ ] Implement `group/welcome.ts`: set/off welcome, subscribe `group.join`, replace `{user}` and `{group}`.
- [ ] Test DB state changes and ctx replies.
